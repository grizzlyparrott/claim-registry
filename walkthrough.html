What verify checks

verify recomputes SHA-256 for every file listed in record/manifest.json, recomputes record_json_hash from canonical JSON, verifies prev_record_hash forms a chain through registry.jsonl, and re-derives the deterministic record_id from stable fields. Any mismatch produces a failure with an explicit reason.

Run it

python -m claim_registry verify --root /path/to/unzipped_packet
